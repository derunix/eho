# Eho Dialogue Extractor

Инструмент для извлечения реплик, монологов и фактов о мире Ехо из книг Макса Фрая с помощью локальной LLM и быстрых regex-эвристик.

## Что умеет

- извлекать голос Сэра Макса из `.fb2`, `.fb2.zip` и `.txt`
- собирать факты о мире Ехо в отдельную базу знаний
- генерировать синтетические пары вопрос-ответ для fine-tune
- писать результаты в `output/` по мере обработки, а не только в конце книги
- продолжать обработку после остановки с того же места
- показывать прогресс, ETA текущей книги и ETA всего пайплайна

## Структура проекта

- `extract_dialogues.py` — основной конвейер
- `extract_regex.py` — быстрый локальный extractor голоса без LLM
- `diagnose_llm.py` — вспомогательная диагностика ответов модели

## Требования

- Python 3.11+
- локальная модель с OpenAI-совместимым API
- для дефолтного сценария: установленный `ollama`

Python-зависимости:

```bash
pip install -r requirements.txt
```

## Быстрый старт

1. Создай и активируй виртуальное окружение.
2. Установи зависимости:

```bash
pip install -r requirements.txt
```

3. Положи книги в папку `books/`.
4. Запусти локальную модель, например:

```bash
ollama run gemma4:e4b
```

5. Запусти пайплайн:

```bash
python extract_dialogues.py
```

Пример с полезными флагами:

```bash
python extract_dialogues.py --workers 2 --voice-extractor regex --chunk-size 2500
```

## Что появляется в `output/`

- `voice_<book>.jsonl` — пары для обучения голоса
- `knowledge_<book>.jsonl` — потоковые факты по мере обработки
- `knowledge_<book>.json` — финальная база фактов по книге
- `synth_<book>.jsonl` — синтетические пары
- `chunks_<book>.jsonl` — checkpoint по обработанным чанкам
- `synth_progress_<book>.jsonl` — checkpoint синтетики
- `dataset.jsonl` — итоговый объединённый датасет
- `knowledge_base.json` — итоговая база знаний

## Возобновление после остановки

Если обработка была прервана, повторный запуск продолжит её с уже сохранённых чанков и фактов. Для этого используются файлы:

- `chunks_<book>.jsonl`
- `synth_progress_<book>.jsonl`
- `done_<book>.marker`

Полностью обработанные книги будут автоматически пропускаться. Для полного пересчёта используй `--no-resume`.
